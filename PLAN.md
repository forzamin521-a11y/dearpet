# DearPet — 애견미용 예약관리 웹앱 개발 계획

> 레퍼런스: app.teepee.kr (스크린샷 15장 + 실제 앱 탐색으로 기능 분석 완료)
> 최종 수정: 2026-07-15

## 1. 확정된 결정사항

| 항목 | 결정 |
|------|------|
| 스택 | Next.js(App Router, TypeScript) + Supabase + Vercel + GitHub |
| UI | Tailwind CSS + shadcn/ui, 화사한 파스텔 퍼플 테마 |
| 알림톡 | v1은 **가짜 발송**(DB에 발송이력만 기록, UI 완성). 발송 모듈을 인터페이스로 분리해 나중에 솔라피 등 API 키만 꽂으면 실발송 |
| 동의서 | 템플릿 편집 + **고객용 모바일 서명 페이지까지** v1 포함 |
| 캘린더 | **담당자별 세로 열 + 일간/주간 뷰** |
| 2차로 미룸 | 이용권(횟수권/금액권), 온라인 예약(고객용), 고객 등급, 쿠폰, 채팅, 알림장, 문자발송 |

## 2. 역할 및 권한 구조

```
슈퍼관리자(제작자, 본인)
 └─ 매장 (가입 신청 → 슈퍼관리자 승인 후 활성화)
     ├─ 매장관리자(원장): 매장 내 모든 권한 + 실장 계정 생성/권한 부여
     └─ 실장(여러 명): 관리자가 이메일/비번/이름으로 계정 생성
         권한 플래그: 예약등록 / 예약변경 / 예약취소 / 예약삭제 / 매출통계 조회
```

- Supabase Auth(이메일+비밀번호) 사용
- 매장 가입 신청 시 `shops.status = 'pending'` → 슈퍼관리자 승인 페이지에서 `approved` 처리
- 실장 계정 생성은 서버 라우트에서 Supabase Admin API로 처리 (관리자가 아이디/비번 입력)
- 모든 테이블에 `shop_id` + RLS(Row Level Security)로 매장 간 데이터 완전 격리

## 3. 데이터베이스 스키마 (Supabase/Postgres)

- **shops** — 매장명, 주소, 전화, 영업시간, 정기휴무, 소개, 로고, status(pending/approved/suspended)
- **profiles** — auth.users 연동. shop_id, role(super_admin/owner/staff), 이름, 전화, 표시색상/이모지, permissions(jsonb), 메모
- **customers** — 보호자 호칭, 전화번호(복수), 알림톡 수신 여부, 메모
- **pets** — customer_id, 이름, 종(강아지/고양이), 품종, 몸무게, 나이/생일, 중성화, 메모
- **grooming_products** — 상품명, 설명 (예: 전체미용, 목욕, 부분+목욕, 스포팅…)
- **product_options** — product_id, 옵션명(소형견 4kg 등), 소요시간, 가격, 적용 몸무게
- **reservations** — customer_id, staff_id(담당자), 날짜, 시작/종료시간, status, 메모, 반복예약 그룹
  - status: `reserved(예약)` `arrived(도착)` `finishing(마무리)` `completed(완료)` `canceled(취소)` `no_show(노쇼)` `deleted(삭제)`
- **reservation_pets** — reservation_id, pet_id, 선택 상품/옵션/추가옵션, 금액
- **sales** — 예약 연동 자동 생성 + 수동 등록/수정. 결제수단별 금액(현금/카드/계좌이체), 판매내역, 담당자, 일자
- **alimtalk_templates** — 유형(기본/노령견/동의서/상황별: 접수·사전·변경·취소·마무리·노쇼), 내용, 사용 여부, 노출 정보 설정
- **alimtalk_logs** — 수신자, 템플릿, 치환된 내용, 상태(`simulated` → 나중에 `sent/failed`)
- **consent_forms** — 동의서 제목/본문(치환변수 {{shopName}} {{dispPet}} 등), 요청 항목
- **consent_submissions** — 서명 토큰(공개 링크용), customer/pet, 서명 이미지(Storage), 서명일시
- **daily_memos** — 날짜별 매장 메모 (캘린더 사이드바)

## 4. 화면 구성

### 4-1. 인증
- 로그인 / 매장관리자 회원가입(매장정보 입력 → 승인 대기 안내)
- 슈퍼관리자 전용: 매장 가입 승인/정지 관리

### 4-2. 예약 캘린더 (핵심 화면)
- 좌측 사이드바: 신규예약 버튼, 미니 달력, 날짜별 메모, 매장명
- 메인: 일간(담당자별 세로 열) / 주간 뷰 전환, 날짜 네비게이션, 검색
- 예약 박스: 보호자호칭, 시간, 반려동물(품종), 상품, 상태별 색상 배지
- 빈 시간 클릭 → 해당 시간이 미리 채워진 신규예약 모달
- **신규예약 모달** (Sheet/Dialog):
  1. 보호자 정보 — 호칭 입력 시 자동완성 `호칭 (전화 뒷4자리) - 펫이름 (품종)`, 선택하면 전화/메모/펫 정보 자동 입력. 신규 고객이면 그대로 입력 → 예약과 동시에 고객 등록
  2. 반려동물 — 자동 로드 + "미용견 추가" (+/- 마리 수)
  3. 예약 정보 — 날짜, 시작시간 선택 + 상품(옵션) 선택 → **소요시간으로 종료시간 자동 계산**, 담당자 지정, 메모
  4. 알림톡 선택 발송 — 기본(디폴트 체크)/노령견/동의서 다중 선택
  5. 동의서 선택
  - [예약 접수] / [예약 접수 후 매출 등록]
- **예약 상세 패널**: 상태 드롭다운(예약→도착→마무리→완료/취소/노쇼/삭제), 보호자/반려동물/이용 정보 탭, 결제 정보
  - 상태변경 시 설정에서 켜둔 상황별 알림톡 자동 발송(가짜)
  - **완료 시 매출 자동 생성** → 결제수단만 선택

### 4-3. 고객 관리
- 검색(호칭/펫이름/전화), 테이블: 프로필, 반려동물명, 보호자호칭, 연락처, 완료/취소/노쇼 횟수
- 우측 패널: 신규 고객 등록(보호자 + 반려동물 N마리), 수정/삭제

### 4-4. 매출 통계 (권한자만)
- 필터: 조회기간(오늘/주/월/직접), 담당자, 상태
- 결제수단 비율 도넛 차트, 결제수단별 합계, 매출 리스트(일자/담당자/보호자/펫/판매내역/금액)
- 매출 수동 등록/수정/삭제, CSV(엑셀) 다운로드

### 4-5. 설정
- 매장 정보 (매장명, 주소, 전화, 영업시간, 휴무, 소개, 로고)
- 미용 상품 관리 (상품 + 세부옵션: 옵션명/소요시간/가격/몸무게)
- 계정 관리 (실장 추가: 이메일/비번/이름/전화/메모, 권한 체크박스)
- 알림톡 설정 (기본/노령견/동의서 템플릿 내용, 상황별 발송 사용 여부, 미리보기)
- 동의서 설정 (동의서 추가/편집, 치환변수, 미리보기)
- 예약 박스 노출 설정 (박스에 표시할 정보 토글)

### 4-6. 고객용 동의서 서명 페이지 (공개, 모바일 최적화)
- `/consent/[token]` — 로그인 불필요, 동의서 내용 + 서명 캔버스 → 제출 시 Storage에 서명 저장
- 서명 완료 시 예약 상세에 "동의서 작성 완료" 표시

## 5. 운영 효율 개선 아이디어 (teepee 대비 +α)

1. **노령견 자동 감지** — 펫 나이 7세 이상이면 예약 모달에서 노령견 알림톡/동의서 자동 체크
2. **동의서 재발송 생략** — 이미 서명한 고객은 동의서 알림톡 기본 해제 (한번 서명하면 계속 적용)
3. **캘린더에서 바로 상태변경** — 예약 박스 우클릭/호버 메뉴로 도착·완료 원클릭
4. **예약 겹침 경고** — 같은 담당자 시간대 겹치면 저장 전 경고
5. **반복 예약** — 4주마다 등 정기 고객 반복 등록
6. **완료 → 매출 원클릭** — 예약에 입력된 상품가 합계가 자동으로 들어가고 결제수단만 탭

## 6. 기술 세부

- **shadcn/ui 주요 컴포넌트**: Sidebar, Dialog, Sheet, Command(자동완성), Calendar, Select, Popover, Table, Card, Tabs, Badge, Switch, Sonner(토스트), Chart(도넛)
- **테마**: 퍼플(#7C3AED 계열) 프라이머리 + 파스텔 배경, 라운드 크게, 화사한 느낌
- **캘린더 그리드**: 라이브러리 없이 CSS Grid 직접 구현 (담당자 열 × 30분 슬롯) — 커스텀 자유도 확보
- **알림톡 모듈**: `lib/messaging/` 에 `MessageProvider` 인터페이스 → `FakeProvider`(v1) / `SolapiProvider`(추후)
- **서명**: canvas 기반 (react-signature-canvas 또는 직접 구현) → Supabase Storage
- **권한**: 미들웨어 + RLS 이중 방어. UI에서도 권한 없는 버튼 비활성화
